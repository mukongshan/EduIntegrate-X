开发与部署注意事项

- 开发环境：本地开发和联调时可以使用 `localhost`（例如 `http://localhost:8081`）。
- 部署环境：上线/部署到测试或生产网络时，请使用内网地址 `10.60.254.43:8081`（例如 `http://10.60.254.43:8081`）。

说明：在开发和部署之间切换时，请注意下列几点：

1. 配置文件：不要将生产凭据或真实连接字符串写死在代码中，使用根目录的 `.env` 或环境变量来管理不同环境的配置（仓库中可保留 `.env.example` 作为示例）。
2. CORS 与防火墙：本地开发通常允许 `localhost`，但部署到 `10.60.254.43` 时可能需要在防火墙或反向代理中放行相应端口（默认 `8081`）。
3. 服务发现：若多个服务部署在同一网段，请把主机名/端口写入部署时的服务配置，不要在代码里硬编码。使用环境变量或配置中心更易于维护。
4. 联调说明：对接各学院系统时，开发时可用 `localhost` 起占位服务（mock）；联调或演示时请切换到 `10.60.254.43:8081` 并通知对方以便在其网段放行。

示例（`.env.example` 中可包含）：

API_BASE_URL=http://localhost:8081 # 开发时替换为 http://10.60.254.43:8081 用于部署/联调
